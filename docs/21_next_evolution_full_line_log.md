# 次ドリンク・進化段階・満杯ライン表示の追加

## 変更日

2026-06-13

## 現状調査

- `index.html` は単一HTMLで、Three.jsを使って3DステージとHUDを描画している。
- 次に発生するオブジェクトは `state.nextLevel` で管理され、射出後に `randomNextLevel()` で更新されている。
- 進化段階は `drinks` 配列に10段階分まとまっている。
- 満杯判定は `isStageFull()` にあり、盤面上の個数が24以上、または手前側の `zAtProgress(0.18)` より前に低速オブジェクトが3個以上ある場合に成立する。

## 変更内容

- HUDに次に発生するドリンクのアイコン、段階、名前を表示した。
- HUDに全10段階の進化表を表示した。
- ステージ上に満杯判定の基準ラインを赤系のラインとして描画した。
- 満杯判定ラインのZ座標を `stageFullLineZ()` にまとめ、判定と描画が同じ値を使うようにした。
- ドリンクのグラデーション生成を `drinkGradient()` にまとめ、表示要素間で重複しないようにした。

## 理由

- プレイヤーが次に発生するオブジェクトを事前に把握できると、配置の判断がしやすくなる。
- 進化段階を常時確認できると、合体後の目標を理解しやすくなる。
- 満杯判定ラインを見えるようにすることで、ゲームオーバー条件の納得感を高められる。

## 確認項目

- `index.html` のHUDにNEXTとEVOLUTIONが表示されること。
- 次ドリンク変更時や射出後にNEXT表示が更新されること。
- ステージ手前側に満杯判定ラインが描画されること。
