# 満杯判定のオブジェクト数条件緩和

## 変更日

2026-06-13

## 現状調査

- `isStageFull()` では、盤面オブジェクト数が24個以上の場合に満杯としていた。
- もう一つの条件として、満杯ラインより手前側に低速オブジェクトが3個以上ある場合にも満杯としていた。
- 24個の数条件は、ラインに触れている感覚より先にゲームオーバーになりやすく、体感として厳しすぎる可能性があった。

## 変更内容

- オブジェクト数による満杯判定を `24` から `100` に変更した。
- 数条件は `safetyStageFullObjectLimit` として定数化し、安全策としての意図を分かりやすくした。
- ライン手前の低速オブジェクト3個以上という判定はそのまま残した。

## 理由

- 基本的には満杯ラインに触れている状態をゲームオーバーの主条件にするため。
- オブジェクト数条件は、極端に盤面数が増えた場合の安全策として残すため。

## 確認項目

- `isStageFull()` のオブジェクト数しきい値が100になっていること。
- ライン手前の低速オブジェクト数による満杯判定が残っていること。
