# 注文対象のタップ提供仕様への変更

## 変更日

2026-06-13

## 現状調査

- 合体で注文対象のオブジェクトが発生した時点で、追加スコアやコインが入り、注文対象が切り替わっていた。
- ステージ上のオブジェクトをタップして提供する入力処理はなかった。
- プレイヤーがどのオブジェクトを提供できるかを盤面上で見分ける表示もなかった。

## 変更内容

- 合体時に注文対象だった場合の自動ボーナス加算と自動注文切り替えを削除した。
- 盤面上の注文対象オブジェクトをタップすると、提供としてオブジェクトを削除し、そのオブジェクトの基本スコアを加算するようにした。
- 提供時に `1 + 進化段階 * 進化段階` の売上を加算するようにした。
- 提供後は新しい注文対象を抽選し、HUDを更新するようにした。
- 注文対象と同じ段階の盤面オブジェクトに、虹色に変化するリング状のハイライトを追加した。
- ハイライトはリングを3本に増やし、線幅と彩度を上げて視認性を高めた。
- ハイライトとタップ判定は、HUDの注文票に実際に表示されている対象と一致するようにした。
- Canvasのタップ時に、提供可能オブジェクトを優先して判定し、提供した場合は射出操作を行わないようにした。

## 理由

- 注文票の達成を、合体時の自動処理ではなくプレイヤーの明示的な提供操作にするため。
- 提供できる対象をステージ上で視認できるようにし、誤操作や見落としを減らすため。

## 確認項目

- 注文対象が合体で発生しても、その瞬間には注文が切り替わらないこと。
- 注文対象の盤面オブジェクトにハイライトが表示されること。
- ハイライトされたオブジェクトをタップすると削除され、スコアが加算されること。
- ハイライトされたオブジェクトをタップすると売上も加算されること。
- 提供後に注文票が新しい対象へ更新されること。
- 対象を提供したタップでは射出が発生しないこと。
