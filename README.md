# NISE SUIKA

スマートフォン縦画面向けのHTMLゲームプロトタイプです。

スイカゲームのように、同じ種類のドリンク同士をぶつけて合体させ、より大きく豪華なドリンクへ進化させます。UnityでAndroid向けゲームを制作する前段階として、操作感、合体の気持ちよさ、3D表示での見やすさを短いサイクルで検証するために作成しています。

## 公開ページ

GitHub Pagesで公開する場合は、リポジトリのPages設定を有効化したあと、以下の形式でアクセスできます。

```text
https://<GitHubユーザー名>.github.io/<リポジトリ名>/
```

トップページは `index.html` です。

## 遊び方

1. タイトル画面で「スタート」を押します。
2. モードを選択します。
3. ステージ手前のドリンクを指で左右にスライドして位置を決めます。
4. 指を離すと、ドリンクが奥方向へ射出されます。
5. 同じ種類のドリンクが接触すると、1段階上のドリンクへ合体します。
6. 目的に応じてスコア、射出数、タイム、注文達成を狙います。

プレイ中は「やめる」ボタンから結果画面へ移動できます。

## ゲームモード

| モード | 内容 |
| --- | --- |
| 無限スコア | ステージが詰まるまで合体を続け、スコアを伸ばします。 |
| 進化チャレンジ | 10段階目を作るまでの射出数の少なさを競います。 |
| 進化タイムアタック | 10段階目を作るまでのタイムを競います。 |
| オーダーラッシュ | 60秒間で注文票のドリンクを作り、ボーナス込みのスコアを競います。 |

## ドリンク進化

| 段階 | 名前 | 得点 |
| --- | --- | ---: |
| 1 | レモンソーダ | 10 |
| 2 | ライムジュース | 25 |
| 3 | ベリーソーダ | 60 |
| 4 | オレンジスカッシュ | 130 |
| 5 | トロピカルジュース | 280 |
| 6 | ブルーハワイ | 560 |
| 7 | ピーチフロート | 1100 |
| 8 | メロンパフェ | 2200 |
| 9 | スターサンデー | 4400 |
| 10 | スペシャルパフェドリンク | 9000 |

最大段階のスペシャルパフェドリンクは、合体で発生したあと盤面に残らず、ボーナスとして扱います。

## 技術構成

- `index.html`: 3D版のメインプロトタイプ
- `index_2d.html`: 2D版の初期プロトタイプ
- `docs/`: 企画メモと実装変更ログ
- Three.js: 3D描画に使用

`index.html` は以下のCDNからThree.jsを読み込みます。

```js
import * as THREE from "https://unpkg.com/three@0.165.0/build/three.module.js";
```

そのため、プレイ時にはインターネット接続が必要です。

## ローカル確認

PowerShellでこのフォルダに移動し、ローカルサーバーを起動します。

```powershell
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

スマートフォン実機で確認する場合は、PCとスマートフォンを同じWi-Fiに接続し、スマートフォンから以下の形式でアクセスします。

```text
http://<PCのIPアドレス>:8000/
```

## 開発メモ

- スマートフォン縦画面でのデモプレイを優先しています。
- パラメーター調整UIは非表示化し、プレイ画面を広く使う構成にしています。
- 物理演算はUnity向けの厳密な剛体シミュレーションではなく、HTMLプロトタイプ用の簡易計算です。
- ドリンクの3Dモデルは外部モデルを使わず、Three.jsの基本ジオメトリを組み合わせて生成しています。
